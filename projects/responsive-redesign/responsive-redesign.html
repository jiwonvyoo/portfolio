Overview

This project presents a full redesign of the MIT Admissions Application Portal, addressing critical usability and accessibility problems identified in the original site. Through a structured design process and responsive HTML/CSS development, the site now adapts seamlessly across screen sizes and user needs.

Part 1: Identifying Usability Problems

Original Website Screenshot

Mobile View

Why I Chose This Site

The MIT Admissions portal is a mission-critical interface that influences thousands of applicants annually. However, it lacks design clarity, especially for new users. For example, the "MIT Admissions" text in the footer is also a hyperlink to the main admissions page, which is not intuitive. It’s public, form-intensive, and significantly underperforms in mobile and accessibility contexts, making it a meaningful candidate for improvement.

Accessibility & Mobile Responsiveness Audit

Using the WAVE Evaluation Tool and Chrome DevTools, I identified major accessibility and responsiveness shortcomings:

  • WAVE: 1 error, 4 alerts, 3 ARIA elements, and 0 contrast issues. The report revealed weak semantic structure and missing assistive technology support.
  • Mobile View: No adaptive layout or spacing. On small screens, links are difficult to tap, content appears cluttered and not focused.

Usability Issues

  • Lack of Clear Buttons: Critical actions are styled as links without visual cues.
  • Poor Information Hierarchy: Key actions are buried in body text without emphasis.
  • Text Readability: Unfriendly font sizes and poor spacing disrupt legibility.

Usability Analysis

Category Problem Impact Solution
Efficiency Primary actions like "Create Account" and "Log In" are not visually prominent and appear as small links. Slows down user action, particularly under time pressure. Increases cognitive load as users scan text for interaction points. Introduce prominent call-to-action buttons with clear labels and primary colors to guide user actions.
Learnability Links are styled inconsistently, and some are embedded within paragraphs without affordances. Users unfamiliar with the site may not recognize clickable areas, leading to frustration or misnavigation. Use consistent styling (e.g., underline and hover effects) for all links and separate important navigation elements from body copy.
Memorability Navigation and layout structure changes between pages, and UI patterns are not reused effectively. Returning users must relearn interface patterns each time, increasing friction and likelihood of abandonment. Design a consistent layout and reusable components that appear across all pages for familiarity and predictability.
Responsiveness Page content does not scale properly on smaller devices; horizontal scroll and visual clutter on mobile. Frustrates users accessing the portal from phones or tablets. This interrupts tasks like submitting forms or reading instructions. Use a responsive grid system and apply flexible images, containers, and media queries to adapt to all screen sizes.

Key Layout Adjustments

  • Used flexbox and CSS grid for scalable, reusable layout components.
  • Defined spacing and visual rhythm through consistent padding and alignment.
  • Created a shared button and card system for clarity and intuitive transitions across viewports.

Part 3: Responsive Redesign

The final site was built in HTML/CSS and deployed to GitHub Pages. It fully implements the mockups and passes the WebAIM WAVE accessibility evaluation with no errors, while adhering to modern responsive design principles. All text, components, and images adapt fluidly from 375px to 1200px+ viewports and respond to font size changes.

Responsiveness Highlights

  • Fully functional and visually consistent on mobile, tablet, and desktop.
  • Responsive to text zoom and long-form content in various languages.
  • Redesign resolves original issues with layout clarity and button visibility.
  • Improved accessibility via color contrast, readable alt text, and semantic tags.
  • Final implementation closely follows mocks and is production-ready.

Reflection

This project taught me to treat accessibility and responsiveness as foundational design principles, not post-hoc additions. The redesign improves usability for all users while reinforcing my skills in layout systems, responsive workflows, and interface clarity. It also gave me practice translating design artifacts into clean, reusable code.
